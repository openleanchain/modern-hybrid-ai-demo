# Security Policy

This repository is an educational reference implementation, not a production service.

Please report suspected vulnerabilities privately to the repository owner rather than opening a public issue. Include the affected component, reproduction steps, impact, and any suggested mitigation.

Do not include API keys, personal information, production data, or other secrets in a report.

The seeded identities and billing records are synthetic. Security reports about missing production infrastructure such as identity-provider integration or managed secret storage are welcome as design feedback but are outside the claim of this demo.
